Фото для демо-каталога (только папки 6 и 7)
=============================================

Куда класть
-----------
public/img/products/6/  — товары продавца 6 (1-й, 3-й, 5-й… в порядке сида)
public/img/products/7/  — товары продавца 7 (2-й, 4-й, 6-й…)

Именованные папки («телефони», «шкафи»…) не нужны — удаляйте их, файлы только в 6/ и 7/.

Имена файлов
------------
Один товар = один timestamp, три варианта v0, v1, v2:

  1779998520_v0_g0_XXXXX.webp
  1779998520_v0_g1_….webp
  1779998520_v1_g0_….webp
  …

Порядок «места» в каталоге (рекомендуется)
------------------------------------------
Добавьте номер в начало — так порядок не собьётся при копировании:

  001_1779998520_v0_g0_XXXXX.webp
  002_1779998650_v0_g0_….webp
  …

Номер 001, 002, 003… — это очередь для ЭТОГО продавца (6 или 7).
Полный список «номер → категория → товар»:
  database/data/catalog_seed_image_order.md

Без префикса 001_ сортировка идёт по timestamp (время в имени).

Что делает migrate:fresh --seed
-------------------------------
1. Удаляет только img_{variant_id}.webp (прошлый вывод сида).
2. Берёт ваши timestamp_*.webp из 6/ и 7/ по номеру (001_, 002_… или по времени).
3. 1-й батч в папке 6 → 1-й товар продавца 6 в каталоге, 2-й батч → 2-й товар и т.д.
4. Копирует в img_{variant_id}.webp — исходники с номером не удаляет.
5. Товары без своего батча — фото из той же категории (без перемешивания).

Нельзя
------
✗ Класть готовые img_42.webp вместо исходников — они пересоздаются
✗ Смешивать батчи продавцов (фото продавца 6 только в папке 6)

После правок: php artisan migrate:fresh --seed
